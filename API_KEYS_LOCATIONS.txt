================================================================================

                            SETUP CHECKLIST
================================================================================

FOR LOCAL DEVELOPMENT:
  □ Create .env.local file
  □ Add VITE_SUPABASE_PROJECT_ID
  □ Add VITE_SUPABASE_URL
  □ Add VITE_SUPABASE_PUBLISHABLE_KEY
  □ Run: npm install
  □ Run: npm run dev

FOR SUPABASE EDGE FUNCTIONS:
  □ Go to Supabase Dashboard → Edge Functions → Settings
  □ Add SUPABASE_URL
  □ Add SUPABASE_ANON_KEY
  □ Add SUPABASE_SERVICE_ROLE_KEY
  □ Add LOVABLE_API_KEY
  □ Add TAVILY_API_KEY (optional)
  □ Deploy all Edge Functions using: supabase functions deploy <function-name>

FOR VERCEL DEPLOYMENT:
  □ Go to Vercel Dashboard → Project → Settings → Environment Variables
  □ Add VITE_SUPABASE_PROJECT_ID
  □ Add VITE_SUPABASE_URL
  □ Add VITE_SUPABASE_PUBLISHABLE_KEY
  □ Set for: Production, Preview, Development
  □ Deploy

                            QUICK REFERENCE

  Dashboard → Settings → General → Copy Reference ID

GET LOVABLE API KEY:
  https://lovable.dev → Settings → API Keys → Create new key

GET TAVILY API KEY:
  https://tavily.com → Dashboard → Get API key

================================================================================

                           SECURITY REMINDERS
================================================================================

✅ DO:
  • Use .env.local for local development
  • Add .env.local to .gitignore
  • Use environment variables in Vercel/hosting platforms
  • Use SUPABASE_PUBLISHABLE_KEY in frontend
  • Use SUPABASE_SERVICE_ROLE_KEY only in Edge Functions
  • Rotate API keys periodically

❌ DON'T:
  • Never commit API keys to Git
  • Never use SERVICE_ROLE_KEY in frontend
  • Never hardcode credentials in source code
  • Never share .env.local file publicly
  • Never commit .env files to version control
